Payments REST API

Dnsc Obs Payments

Use the Payments API to easily and securely accept online and mobile Dnsc Obs payments. You can enable customers to make Dnsc Obs payments with only a few clicks. A completed payment is known as a sale.

Integration steps
1. Required Set up your development environment.
2. Required Create Dnsc Obs payment.
3. Required Execute payment.

Set up your development environment

Before you can integrate Payments, you must set up your development environment. After you get a token that lets you access protected REST API resources, you create sandbox accounts to test your web and mobile apps.

Create Dnsc Obs payment

After you collect the payment details from the customer, specify the payment details in a /payment call.

In the request URI, set the Authorization.

In the JSON request body, set the intent to sale

And the accountnumber to your bank account number

Important: You cannot run the following if you dont replace all call-specific parameter values, such as tokens and IDs, with your own values.
PHP-Sample Code

<?php
error_reporting(0);
function payment($cvc,$cardnumber,$amount) {
$curl = curl_init();
curl_setopt_array($curl, array(
CURLOPT_URL => "http://127.0.0.1:8888/payment",
CURLOPT_RETURNTRANSFER => true,
CURLOPT_ENCODING => "",
CURLOPT_MAXREDIRS => 10,
CURLOPT_TIMEOUT => 0,
CURLOPT_FOLLOWLOCATION => true,
CURLOPT_HTTP_VERSION => CURL_HTTP_VERSION_1_1,
CURLOPT_CUSTOMREQUEST => "GET",
CURLOPT_HTTPHEADER => array(
    "API-KEY: ",
    "intent: Sale",
    "cvc:".$cvc."",
    "cardnumber:".$cardnumber."",
    "amount:".$amount."",
    "accountnumber:"
  ),
));
$response = curl_exec($curl);
curl_close($curl);

return $response;
}

?>

Error List

The api will return the following error



{"Msg":"API KEY ERROR","status":False,"errcode":401} - Returns if the Api key is invalid

{"Msg":"Intent Error, Please Set your intent into 'Sale' ","status":False,"errcode":402} - Returns if the intent is not sale

{"Msg":'Non-sufficient funds!',"status":False,"errcode":412} - Returns if the user has not enough funds in the account

{"Msg":"Credit Card Credentials Error","status":False,"errcode":405} - Returns if the card details is invalid
